# 员工晋升管理制度

> 文件编号：HR-2024-017  
> 版本号：V2.8  
> 生效日期：2024年2月1日  
> 审批人：首席执行官（CEO）、人力资源总监  
> 适用范围：某科技有限公司全体正式员工

---

## 概述

为建立公平、透明、有序的员工职业发展通道，激发员工成长动力，某科技有限公司（以下简称"公司"）制定本**晋升**管理制度。本制度规定了员工晋升的基本条件、晋升流程、答辩评审标准及晋升后的薪酬调整规则，适用于公司全体正式员工。

公司实行双通道职业发展体系——**专业序列（P线）** 和**管理序列（M线）**，两条路径同级别薪酬对等，员工可根据自身兴趣和能力特长在两条通道间切换发展。

---

## 政策标准

### 晋升周期

公司每年开放**2次晋升窗口**：

| 窗口 | 提名月 | 答辩月 | 生效月 |
|------|--------|--------|--------|
| 春季窗口 | 2月 | 3月 | 4月 |
| 秋季窗口 | 8月 | 9月 | 10月 |

晋升**答辩**前一个月开放提名通道，HR将发送全员通知邮件，说明当次晋升窗口的时间安排和材料提交要求。

### 晋升条件

员工满足以下条件之一即可申请晋升：

1. 在当前岗位**任职满1年**（从最近一次晋升/入职转正日期起计算）
2. **连续2次**绩效评估结果为B+及以上（年度/半年度考核）
3. 主导完成公司级重点项目并获得项目评审委员会认可
4. 通过内部竞聘获得新岗位（新岗位级别高于现岗位）

原则上不允许跨级晋升（例如从P2直接升至P4），特殊情况须经晋升评审委员会主任委员特批。

### 职业发展通道

| 序列 | 级别 | 对应职务 | 薪酬带宽 |
|------|------|----------|----------|
| P线 | P1 | 初级工程师/专员 | 初级 |
| P线 | P2 | 中级工程师/高级专员 | 中级 |
| P线 | P3 | 高级工程师/资深专员 | 中高级 |
| P线 | P4 | 资深工程师/专家 | 高级 |
| P线 | P5 | 专家 | 专家级 |
| P线 | P6 | 资深专家 | 资深专家级 |
| P线 | P7 | 科学家 | 最高级 |
| M线 | M1 | 团队主管 | 中高级 |
| M线 | M2 | 部门经理 | 高级 |
| M线 | M3 | 总监 | 专家级 |
| M线 | M4 | VP | 资深专家级 |
| M线 | M5 | CXO | 最高级 |

P线与M线在同等薪酬带宽内保持对等（如P4与M2薪酬带宽基本一致）。员工可在晋升申请时选择目标序列和级别，由评审委员会评估是否满足条件。

## 申请流程

### 晋升流程

```
提名 → 提交材料 → 晋升答辩 → 评审公示 → 晋升生效
```

1. **提名**：直属领导提名或员工自荐。提名时须填写《晋升提名表》，说明提名理由及目标级别。
2. **提交材料**：被提名人提交晋升材料包，包含：
   - 《工作成果总结》（不少于800字，含量化数据）
   - 《未来工作规划》（不少于500字，含团队/技术发展规划）
   - 近2次绩效考核结果
   - 推荐信（至少1封，直属领导或跨部门合作负责人）
3. **晋升评审答辩**：由晋升评审委员会（通常3-5人组成）组织答辩，形式为：
   - 20分钟个人陈述（工作成果 + 技术/管理能力展示 + 未来规划）
   - 10分钟评审委员Q&A
4. **评审结果公示**：答辩结束后5个工作日内，HR将评审结果在所有评审委员签字确认后公示（仅公示通过名单），公示期为5个工作日。
5. **晋升生效**：公示期无异议的，晋升自当次晋升窗口的生效月起生效，**薪酬调整同步进行**。

### 晋升评审委员会

评审委员会由以下人员组成（根据被评人级别调整）：

- 主任委员：1人（被评人所在部门的上一级负责人）
- 委员：2-4人（跨部门相关领域专家或管理者）
- 列席：HR代表（无投票权，负责流程合规记录）

---

## 注意事项

1. 晋升提名材料须真实、完整。如发现材料造假，取消当次及下次晋升资格，并记入个人诚信档案。
2. 公示期内如有实名异议，HR须在5个工作日内调查并反馈调查结果。
3. 晋升未通过者可参加下一次晋升窗口，同一级别连续3次未通过的，须间隔一年后方可再次申请。
4. 晋升后试用期为3个月（内部称"考察期"），考察期内如发现不胜任新岗位，公司有权恢复原级别和薪资。
5. 管理序列（M线）晋升须考察管理能力，首次申请管理序列的员工须完成公司内部《基层管理能力培训》课程。
