# 公司信息安全管理制度

> 版本：V3.2 | 生效日期：2026-01-01 | 发布部门：信息安全委员会

---

## 一、总则

### 1.1 目的

为保障公司信息系统及数据的机密性、完整性、可用性，规范员工信息安全行为，防范信息泄露和安全事件，特制定本制度。

### 1.2 适用范围

本制度适用于 KnowBrain 公司全体员工（含正式员工、实习生、外包人员、驻场顾问）。离职后保密义务继续有效。

### 1.3 信息安全等级

| 等级 | 标识 | 定义 | 示例 |
|------|------|------|------|
| L1 公开 | 绿色 | 可对外公开的信息 | 产品介绍、招聘信息 |
| L2 内部 | 黄色 | 仅限公司内部知晓 | 内部流程制度、培训资料 |
| L3 机密 | 橙色 | 仅限相关部门知晓 | 未发布产品方案、合同、薪资 |
| L4 绝密 | 红色 | 仅限特定授权人员 | 源代码核心模块、客户隐私数据、加密密钥 |

## 二、账号与密码管理

### 2.1 密码策略

| 要求 | 说明 |
|------|------|
| 最小长度 | 10 位 |
| 复杂度 | 必须包含大写字母、小写字母、数字、特殊字符中至少 3 类 |
| 修改周期 | 域密码每 90 天强制修改 |
| 历史限制 | 不能与最近 5 次使用过的密码相同 |
| 初始密码 | 首次登录必须修改 |
| 禁止行为 | 禁止使用生日、工号、姓名拼音等易猜测组合 |

### 2.2 多因素认证（MFA）

以下系统必须启用双因素认证：

- 代码仓库（GitLab）
- 财务系统
- VPN 远程接入
- 生产服务器 SSH
- 云服务控制台（阿里云/AWS）

MFA 工具：企业微信内置「身份验证器」或 Google Authenticator。

### 2.3 账号管理

- 每个账号仅限本人使用，**禁止共享账号**
- 离职/转岗时 IT 在当天 18:00 前回收权限
- 超过 90 天未登录的账号自动禁用
- 外包/驻场人员账号标注临时标识，到期自动失效

## 三、数据安全

### 3.1 数据分类处理要求

| 等级 | 存储 | 传输 | 共享 | 销毁 |
|------|------|------|------|------|
| L1 公开 | 无限制 | 无限制 | 无限制 | 普通删除 |
| L2 内部 | 公司设备或云盘 | 公司邮箱/IM | 内部可查看 | 安全删除 |
| L3 机密 | 加密存储 | 加密传输 | 最小权限审批 | 安全擦除 |
| L4 绝密 | 隔离环境 + 加密 | 专用加密通道 | 逐项审批 | 物理销毁 |

### 3.2 数据备份

- 文件服务器（NAS）：每日增量备份，每周全量备份，保留 90 天
- 数据库：每日自动备份，保留 30 天
- 邮件：云端实时备份，保留 365 天
- 员工本地数据：**不纳入公司备份范围**，重要文件需存入 NAS 或 OneDrive

### 3.3 数据传输

- 发送含 L3 级以上信息的文件，必须使用加密压缩（密码通过另一渠道告知）
- 禁止使用个人网盘（百度网盘/Google Drive 个人版）存储公司数据
- 公司提供 OneDrive for Business 作为云存储工具

## 四、设备安全

### 4.1 办公电脑安全

| 要求 | 说明 |
|------|------|
| 全盘加密 | 所有办公电脑必须启用 BitLocker（Windows）或 FileVault（macOS） |
| 锁屏策略 | 离开工位必须锁屏（Win+L / Ctrl+Cmd+Q），超时 5 分钟自动锁屏 |
| 防病毒 | 统一安装 CrowdStrike Falcon，IT 后台监控安全事件 |
| 系统更新 | 操作系统安全补丁在发布后 7 天内强制推送安装 |
| 软件安装 | 仅允许从公司软件中心（Software Center）安装，禁止自行下载破解软件 |

### 4.2 移动设备管理（MDM）

- 访问公司邮件/IM 的个人手机必须注册 MDM（Microsoft Intune）
- MDM 不收集个人隐私数据（照片、位置、浏览记录）
- 设备丢失需在 2 小时内报告 IT，IT 远程擦除公司数据
- 离职时 IT 协助移除 MDM 配置文件

### 4.3 可移动存储

- U 盘和移动硬盘默认只读（可通过 IT 申请临时写入权限）
- 严禁使用来路不明的 U 盘插入公司电脑
- 遗失含公司数据的移动存储设备需立即上报

## 五、网络安全

### 5.1 网络访问控制

- 办公网与访客网物理隔离
- 生产服务器网段与办公网段防火墙隔离
- 外部访问内网系统必须通过 VPN
- 禁止私自搭建 WiFi 热点或使用随身 WiFi

### 5.2 互联网使用规范

| 允许 | 禁止 |
|------|------|
| 工作相关网站和应用 | 赌博、色情、非法网站 |
| 技术社区（GitHub/Stack Overflow） | P2P 下载（BT/迅雷） |
| 行业资讯和学术资源 | 翻墙工具、代理软件 |
| 即时通讯（企业微信/微信） | 未经批准的直播/视频上传 |

### 5.3 远程办公安全

- 远程办公必须通过 VPN 连接公司网络
- 禁止在公共 WiFi（咖啡馆、机场）不经过 VPN 访问内部系统
- 远程会议涉及屏幕共享时，注意关闭非相关工作窗口和 IM 通知
- 纸质机密文件不得带离办公场所

## 六、安全事件响应

### 6.1 事件分类

| 等级 | 定义 | 响应时效 | 示例 |
|------|------|---------|------|
| P1 紧急 | 核心系统中断 / 数据泄露 | 15 分钟 | 勒索病毒、源代码泄露 |
| P2 严重 | 非核心系统中断 / 钓鱼攻击 | 1 小时 | 钓鱼邮件批量发送、WiFi 瘫痪 |
| P3 一般 | 单设备安全告警 | 4 小时 | 病毒告警、异常登录 |
| P4 低危 | 潜在风险 | 24 小时 | 弱密码提醒、证书即将过期 |

### 6.2 报告流程

1. 发现安全事件 → 立即通知部门负责人 + IT 运维（分机 8888）
2. 如涉及数据泄露 → 同时通知信息安全委员会（`security@knowbrain.com`）
3. IT 初步判断事件等级 → 按响应时效启动应急处理
4. 事件处理后 48 小时内提交《安全事件分析报告》
5. **关键原则**：宁可误报，不要瞒报

### 6.3 违规处罚

| 违规等级 | 行为示例 | 处罚 |
|---------|---------|------|
| 轻微 | 离开工位未锁屏（首次） | 口头提醒 |
| 一般 | 使用弱密码、私自安装未授权软件 | 书面警告 |
| 严重 | 共享账号、敏感数据未加密外发 | 记过 + 扣绩效 |
| 重大 | 故意泄露机密数据、越权访问客户隐私 | 解除劳动合同 + 追究法律责任 |

## 七、员工安全意识培训

- **入职培训**：入职当天完成信息安全必修课（约 30 分钟）
- **年度培训**：每年 6 月完成信息安全复训
- **钓鱼演练**：每季度 IT 发送模拟钓鱼邮件，点击率纳入部门安全评分
- **通报机制**：真实安全事件（脱敏后）每月安全月报通报全员
