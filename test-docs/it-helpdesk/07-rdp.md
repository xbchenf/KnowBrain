# 远程桌面与远程办公技术管理规定

> 文档版本：v2.1 | 生效日期：2025-01-20 | 责任部门：IT 基础设施部 / 人力资源部 | 适用范围：某科技有限公司全体正式员工（实习生及外包人员须单独申请）

---

## 概述

随着混合办公模式的常态化，某科技有限公司为员工提供了规范的远程桌面接入方案，使员工在居家办公、出差等场景下能够安全访问公司办公电脑上的文件、应用和内网系统资源。

远程桌面服务基于 Windows Remote Desktop Service（RDP）协议，结合公司 VPN 加密通道保障传输安全。此外，Microsoft Teams 提供了轻量级的远程控制功能，适用于临时性、短时间的远程协助场景。

**重要前提**：远程办公须提前申请权限。员工通过企业微信「OA 审批 → 远程办公权限申请」提交申请，经**直属领导审批**通过后，由 IT 部门在 1 个工作日内开通。未经审批擅自使用远程桌面工具接入公司网络的，视为违反信息安全管理制度。

## 方式一：VPN + 远程桌面（RDP，推荐）

本方式适用于需要在家庭电脑上完整操作公司办公电脑的场景，体验接近在办公室直接使用电脑。

### 前置准备（在办公室完成）

在离开办公室之前，须在公司电脑上完成以下一次性设置：

1. 启用远程桌面功能：
   - 打开「设置」→「系统」→「远程桌面」
   - 将「启用远程桌面」开关切换为**开启**
   - 系统会自动添加当前登录的企业账号到远程桌面用户组
2. 记录公司电脑的计算机名：
   - 「设置」→「系统」→「关于」→ 查看「设备名称」（如 `DESKTOP-ZH00123`）
   - 或者在命令提示符中运行 `hostname` 命令查看
   - 将此计算机名保存到手机备忘录或拍照保存
3. 确认公司电脑的电源设置：
   - 「设置」→「系统」→「电源和睡眠」→ 将「睡眠」设置为「从不」
   - 确保电脑保持开机状态且处于联网状态（不要关机，可以合上笔记本盖子但须在电源选项中将合盖操作设为「不采取任何操作」）
4. 记录公司电脑的 IP 地址作为备用：命令提示符运行 `ipconfig`，记录 IPv4 地址（如 192.168.1.105）

### 远程连接步骤（在家庭电脑上操作）

1. **先连接公司 VPN**：按照《VPN 远程接入配置与管理规范》（01-vpn.md）完成 VPN 连接建立
2. VPN 连接成功后，按键盘 **Win + R** 打开运行窗口，输入 `mstsc`，回车打开远程桌面连接程序
3. 在「计算机」输入框中输入公司电脑的**计算机名**或 **IP 地址**
4. 点击「显示选项」展开详细设置：
   - 「显示」标签：根据家庭电脑屏幕分辨率调整远程桌面窗口大小，建议选择「全屏」
   - 「本地资源」标签 →「远程音频」：选择「在远程计算机上播放」
   - 「本地资源」标签 →「剪贴板」：勾选以允许本地和远程电脑之间复制粘贴文本
   - 「体验」标签：根据家庭网络质量选择连接速度（默认「自动检测」即可）
5. 点击「连接」，在弹出的登录窗口中输入企业账号和密码
6. 如弹出证书警告，勾选「不再询问」并点击「是」
7. 等待连接建立，成功后屏幕将显示公司电脑的桌面环境

### 断开与重新连接

- 关闭远程桌面窗口（点击窗口右上角 X）不会注销远程电脑的会话，下次可快速恢复
- 建议下班时在远程电脑上正常「注销」（开始菜单 → 用户图标 → 注销），释放服务器资源

## 方式二：Teams 远程控制

本方式适用于同事之间临时提供远程协助，或需要在另一台已开机的设备上进行轻量操作。

1. 在公司电脑和家庭电脑上均登录同一企业 **Microsoft Teams** 账号
2. 在公司电脑上打开 Teams → 在聊天搜索框中搜索自己的名字 → 点击进入「与自己聊天」的对话窗口
3. 在对话输入框上方点击「共享屏幕」按钮，选择一个屏幕或窗口进行共享
4. 共享开始后，在屏幕共享工具栏或会议控件中点击「授予控制权」→ 在下拉列表中选择你的另一个登录会话
5. 在家庭电脑上确认接受控制权，即可远程操作公司电脑

> 限制说明：Teams 远程控制不支持传输文件、无法使用剪贴板同步，操作延迟高于 RDP。如需长时间办公，建议使用方式一（VPN+RDP）。

## 远程办公注意事项

### 安全要求

1. 家庭电脑须安装并开启防火墙和杀毒软件，操作系统的安全补丁须更新至最新
2. 禁止在公共电脑（如网吧、酒店商务中心）上进行远程桌面连接
3. 远程办公期间不得将公司电脑上的文件下载到家庭电脑本地存储。确有需要的，须通过公司审批后的云盘（OneDrive for Business）进行文件传输
4. 远程桌面会话空闲超过 30 分钟时，系统将自动锁定屏幕，须重新输入密码解锁
5. 禁止将远程桌面凭据（账号密码）保存在家庭电脑的「记住凭据」中

### 性能与体验

1. 远程桌面流畅度主要依赖家庭网络的上行带宽，建议使用有线网络而非 WiFi，最低带宽要求 2 Mbps 上行
2. 远程桌面环境下不建议运行大型开发工具（如 Visual Studio、Android Studio 等），编译和调试延迟较大
3. 如需远程参加视频会议，建议直接在家庭电脑上登录 Teams，而非通过远程桌面（双重编解码会导致音视频质量严重下降）

## 常见问题

| 问题 | 解决办法 |
|------|---------|
| 提示「找不到计算机」 | 确认 VPN 已连接；尝试用 IP 地址代替计算机名；确认公司电脑未关机或进入休眠 |
| 连接后黑屏，只有鼠标指针 | 公司电脑可能处于睡眠状态，可尝试通过 Wake-on-LAN 唤醒（需提前在 BIOS 中开启该功能并联系 IT 配置），或联系在办公室的同事帮忙按一下键盘唤醒电脑 |
| 连接被拒绝，提示「用户未被授权」 | 公司电脑远程桌面用户组未包含你的账号，请联系 IT 服务台协助添加 |
| 远程桌面操作非常卡顿 | 在网络体验设置中将连接速度从「自动检测」改为「低速宽带(256 kbps - 2 Mbps)」，可降低桌面特效和字体平滑以提升响应速度 |

## 权限申请与回收

| 情形 | 处理方式 |
|------|---------|
| 首次申请远程办公权限 | 企业微信 OA 审批 → 直属领导审批 → IT 开通（1 个工作日） |
| 权限到期续期 | 每次审批有效期为 3 个月，到期前 7 天系统提醒续期 |
| 离职/转岗 | HR 通知 IT 后，离职员工远程权限立即回收；转岗员工按新部门重新申请 |
| 安全事件 | 信息安全部有权在安全事件发生时紧急暂停任何员工的远程办公权限，事后补办手续 |
