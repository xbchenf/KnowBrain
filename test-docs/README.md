# KnowBrain 评测测试文档

本目录包含用于 RAG 评测体系的种子测试文档和初始化脚本。

---

## 一键初始化

```bash
# 1. 确保 KnowBrain 服务已启动
docker-compose up -d

# 2. 运行初始化脚本
cd test-docs
node provision.mjs

# 3. 初始化后自动运行基线评测
RUN_EVAL=true node provision.mjs
```

### 脚本做了什么

| 步骤 | 操作 | 产出 |
|------|------|------|
| ① | 管理员登录 | JWT Token |
| ② | 创建 2 个知识空间 |「IT 运维测试库」「HR 制度测试库」|
| ③ | 批量上传 34 篇测试文档 | 每篇文档 → 解析 → 分块 → 向量化入库 |
| ④ | 创建 2 个评测数据集 |「IT 运维回归测试」「HR 制度回归测试」|
| ⑤ | 从 FAQ 导入 34 个评测问题 | 含 question + expectedAnswer |
| ⑥ | (可选) 触发基线评测 | 自动运行评测，生成基线分数 |

### 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `KB_BASE_URL` | `http://localhost:8080` | KnowBrain 服务地址 |
| `KB_ADMIN_USER` | `admin` | 管理员账号 |
| `KB_ADMIN_PASS` | `KnowBrain@2026` | 管理员密码 |
| `RUN_EVAL` | `false` | `true` 则在初始化后自动触发基线评测 |
| `KB_BATCH` | `3` | 文档上传并发数（建议 3-5） |

### 前置条件

1. KnowBrain 服务已启动且可访问
2. 管理员账号已存在（默认 admin/KnowBrain@2026）
3. Node.js 18+ 可用

### 可重复执行

脚本是幂等的 — 每次执行会创建新的空间、数据集和运行记录，不会覆盖已有数据。

## 文档生成规范

每篇文档须满足以下要求：

1. **格式**: Markdown，UTF-8 编码
2. **标题**: 使用 `# 一级标题`，标题须包含文档主题关键字
3. **结构**: 至少包含 3 个二级段落（`##`），如：概述、操作步骤、注意事项、常见问题、相关政策等
4. **内容**: 以 FAQ 答案为骨架，扩展为完整的制度文档或技术手册风格
5. **长度**: 每篇 300-800 字（中文），信息密度适中
6. **真实感**:
   - 加入模拟的公司名称占位（如「某科技有限公司」）
   - 加入合理的上下文段落（背景说明、适用范围、例外情况等）
   - 加入少量无关但合理的内容（模拟真实文档的噪音）
7. **可检索性**: 文档标题和内容须包含 FAQ 中的 keywords，确保关键词检索能命中

## 与 FAQ 的对应关系

每篇测试文档的核心内容对应 `knowbrain-server/src/main/resources/scenarios/{scenario}/faq.json` 中的一条 FAQ：
- FAQ 的 `question` → 文档应该能回答的问题
- FAQ 的 `answer` → 文档的核心段落（不能原样照搬，需扩展）
- FAQ 的 `keywords` → 文档中应自然出现的术语
- FAQ 的 `category` → 文档的文件名

## 文件命名

`{序号}-{category}.md`，序号从 01 开始，category 使用 FAQ 中的 category 字段。

## 对抗测试

### 一键导入

```bash
cd test-docs
node adversarial.mjs                # 仅导入对抗问题
RUN_EVAL=true node adversarial.mjs  # 导入后自动运行评测
```

### 对抗问题设计

对抗测试覆盖 5 类边界场景，每场景 3 题，共计 15 题/场景：

| 类别 | 说明 | 测什么 |
|------|------|--------|
| **跨文档综合** | 需多篇文档信息整合才能完整回答 | 多文档检索覆盖 + 信息整合 |
| **无答案问题** | 文档库中不存在相关信息 | 诚实拒绝，不编造（Faithfulness=1 且 Recall=0） |
| **口语化/术语混用** | 口语表达 + 专业术语混合 | 口语理解 + 术语映射 |
| **模糊简短** | 单关键词或极短查询（1-3 字） | 歧义处理 + 意图推断 |
| **多义词** | 同一表述在不同上下文有不同含义 | 上下文消歧能力 |
