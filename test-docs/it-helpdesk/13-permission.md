# 文件共享与访问权限管理制度

> 版本：V2.2 | 生效日期：2025-03-15 | 适用范围：某科技有限公司全体员工
> 制定部门：IT 基础设施部 | 权限问题支持：分机 8004

## 概述

某科技有限公司使用 Windows Server 域控体系管理文件共享权限。共享文件夹（网络驱动器）按部门、项目组和职能角色划分访问权限。本制度说明权限申请流程、常见拒绝访问（Access Denied）问题的解决方法以及权限管理规范。

合理设置和管理文件夹权限，是保障公司信息安全、防止数据泄露的第一道防线。请勿为图方便将敏感文件夹设置为「Everyone 完全控制」。

## 共享文件夹体系

| 网络驱动器 | 路径 | 用途 | 权限模型 |
|-----------|------|------|---------|
| Z 盘 | `\\fileserver\shared` | 部门共享文件、公司级制度文档 | 部门安全组 + 职能角色 |
| Y 盘 | `\\fileserver\projects` | 项目文件、跨部门协作 | 项目组成员 |
| X 盘 | `\\fileserver\personal` | 个人网络存储（50GB 配额） | 仅本人访问 |

## 权限申请流程

### 标准申请（处理周期：1 个工作日）

1. 访问 IT 自助平台：**https://itsm.company.com**
2. 选择「权限申请」→ 填写以下信息：
   - 需要访问的资源路径（如 `\\fileserver\shared\财务部\2025预算`）
   - 申请权限类型（只读 / 读写 / 完全控制）
   - 申请原因（如：新入职、项目协作、临时查阅）
3. 提交后系统自动发送审批通知给资源负责人或部门助理。
4. 审批通过后，**注销当前 Windows 会话并重新登录**，使新权限生效。

### 紧急权限申请（处理周期：2 小时内）

如因业务紧急需要在 2 小时内获取权限：

1. 发送邮件至直属领导，说明需求并抄送 IT 服务台（itservice@company.com）。
2. 直属领导邮件批复后，IT 可先行开通权限。
3. 事后在自助平台补交正式申请。

## 拒绝访问（Access Denied）常见原因与处理

员工在访问共享文件夹时遇到「拒绝访问」提示，通常有以下原因：

| 原因 | 处理方法 |
|------|---------|
| **新入职 / 部门调动** | 权限尚未同步到域控安全组（AD 同步周期为 2 小时）。等待同步后再试；超时未生效联系 IT。 |
| **网络驱动器映射过期** | 断开网络驱动器后重新映射。打开「此电脑」→ 计算机 → 映射网络驱动器，输入路径和凭据。 |
| **权限组未包含所在部门** | 联系资源负责人确认部门安全组是否已添加。如为新成立部门，需 IT 先创建部门安全组。 |
| **无权访问该资源** | 确认确实需要使用该资源，走标准权限申请流程。 |
| **DNS 解析问题** | 尝试用 IP 地址替代服务器名称访问（如 `\\10.0.xx.xx\shared`），如 IP 可通则联系 IT 检查 DNS。 |

## 权限管理规范（面向资源负责人）

- 新增成员时使用安全组添加，而非单独添加个人账号。
- 员工离职或调岗时，及时从安全组中移除。
- 敏感文件夹（含财务数据、人力资源数据、客户信息）每季度进行一次权限审计，清理不再需要的访问权限。
- 禁止将「Everyone」或「Domain Users」赋予写入/完全控制权限。

## 常见问题

**Q: 文件夹权限多久同步一次？**  
A: AD 域控每 2 小时自动同步一次。紧急权限可请求 IT 手动触发同步。

**Q: 外包人员或访客能否访问共享文件夹？**  
A: 外包人员可通过「外部协作账号」申请有限访问权限。访客不开放共享文件夹访问。
