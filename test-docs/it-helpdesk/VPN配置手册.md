# 公司 VPN 配置手册

> 适用版本：VPN Client v4.2 | 更新日期：2026-06-15 | 维护部门：IT 运维部

---

## 一、概述

公司 VPN（虚拟专用网络）用于员工在办公网络以外（远程办公、出差、居家）安全访问公司内部系统，包括：

| 系统 | 内网地址 | 用途 |
|------|---------|------|
| OA 办公系统 | http://oa.knowbrain.local | 流程审批、公告通知 |
| 代码仓库 (GitLab) | http://gitlab.knowbrain.local | 代码管理 |
| 文件服务器 (NAS) | \\\\nas.knowbrain.local | 共享文件存储 |
| 测试环境 | http://test.knowbrain.local | 开发测试 |
| 财务系统 | http://finance.knowbrain.local | 报销、预算管理 |

## 二、申请流程

1. **提交申请**：登录 OA 系统 → IT 服务 → VPN 权限申请，填写申请原因和有效期
2. **部门审批**：部门负责人审批（1 个工作日内）
3. **IT 开通**：IT 运维在 1 个工作日内完成账号开通，邮件通知
4. **账号有效期**：默认 1 年，到期前 30 天系统自动提醒续期

> ⚠️ **注意**：实习生默认不开通 VPN 权限，如需远程访问请联系直属上级走特殊申请流程。

## 三、客户端安装

### 3.1 下载地址

- **Windows**：http://download.knowbrain.local/vpn/KBVPNClient-4.2-win64.exe
- **macOS**：http://download.knowbrain.local/vpn/KBVPNClient-4.2-mac.dmg
- **Linux**：http://download.knowbrain.local/vpn/KBVPNClient-4.2-linux.tar.gz

### 3.2 Windows 安装步骤

1. 下载 Windows 客户端安装包
2. 右键点击安装包 → **以管理员身份运行**
3. 安装向导选择语言 → 中文（简体）
4. 安装路径保持默认 `C:\Program Files\KnowBrain\VPN\`
5. 勾选「创建桌面快捷方式」
6. 点击「安装」，等待完成
7. 安装完成后**重启电脑**（首次安装必须重启）

### 3.3 macOS 安装步骤

1. 下载 macOS DMG 安装包
2. 双击挂载 DMG 文件
3. 将 `KBVPNClient.app` 拖入 `Applications` 文件夹
4. 首次打开时，在「系统偏好设置 → 安全性与隐私」中点击「仍要打开」
5. 按照提示安装系统扩展（需要输入 Mac 登录密码）

### 3.4 手机端配置

| 平台 | 方式 |
|------|------|
| iOS | App Store 搜索「KBVPN」或使用企业证书安装 |
| Android | 内部应用商店下载或从 http://m.knowbrain.local 下载 APK |

## 四、连接配置

### 4.1 首次配置

1. 打开 KBVPNClient，点击右上角 **+** 号 → 添加连接
2. 填写以下参数：

| 参数 | 值 |
|------|-----|
| 连接名称 | 自定义（如：公司VPN-正式） |
| 服务器地址 | `vpn.knowbrain.com` |
| 端口 | `8443` |
| 协议 | SSL VPN |
| 认证方式 | LDAP 账号认证 |

3. 点击「保存」

### 4.2 登录认证

1. 双击已保存的连接
2. 输入**域账号**（格式：`knowbrain\员工号`，如 `knowbrain\KB00123`）
3. 输入**域密码**（同开机密码）
4. 如需绑定手机令牌，输入 6 位动态验证码
5. 点击「连接」

### 4.3 连接状态说明

| 状态图标 | 含义 |
|---------|------|
| 🟢 绿色 | 已连接，网络正常 |
| 🟡 黄色 | 连接中/重连中 |
| 🔴 红色 | 未连接/连接失败 |
| 🔵 蓝色 | 已连接但无网络流量 |

## 五、常见问题排查

### Q1：提示「无法连接到服务器」

**可能原因**：
- 检查电脑是否能正常上网（浏览器打开百度测试）
- 检查服务器地址是否填写正确：`vpn.knowbrain.com`（不是 `vpn.knowbrain.local`）
- 公司防火墙可能限制了 VPN 端口，切换协议为 SSL VPN over HTTPS（端口 443）

### Q2：提示「账号或密码错误」但密码确认无误

- 域账号密码每 **90 天**强制修改，可能已过期
- 尝试登录公司任意一台内网电脑验证账号状态
- 联系 IT 运维（分机 8888）重置密码

### Q3：连接成功但无法访问内网系统

1. 检查 VPN 客户端是否开启了「全隧道模式」：设置 → 隧道模式 → 选择「Split Tunnel（分离隧道）」
2. 清除 DNS 缓存：
   - Windows：命令行执行 `ipconfig /flushdns`
   - macOS：终端执行 `sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder`
3. 确认内网地址正确（见第一节对照表）

### Q4：频繁掉线

- 检查网络环境稳定性，切换到有线网络测试
- WiFi 信号弱时容易断开，建议信号强度 ≥ -65 dBm
- VPN 客户端设置 → 勾选「自动重连」
- 如持续掉线，IT 建议使用备用协议 WireGuard（端口 51820）

### Q5：手机端 VPN 耗电快

- iOS：设置 → 通用 → 后台应用刷新 → 关闭 KBVPN
- Android：设置 → 电池 → 应用省电策略 → KBVPN → 无限制
- 不需要时建议断开 VPN 连接

## 六、安全要求

| 要求 | 说明 |
|------|------|
| 密码强度 | 域密码长度 ≥ 10 位，含大小写字母 + 数字 + 特殊字符 |
| 密码修改周期 | 每 90 天强制修改 |
| 二次验证 | 访问财务系统、代码仓库必须绑定手机令牌（MFA） |
| 会话超时 | 连续无操作 2 小时自动断开 VPN |
| 禁止行为 | 禁止将 VPN 账号借给他人使用，违者按信息安全制度处罚 |
| 离职回收 | 员工离职当天 IT 自动回收 VPN 权限 |
